Temas del Tercer Parcial

Resumen de temas vistos en clase

En este parcial se aprendió a usar herramientas de desarrollo de aplicaciones sin código como AppSheet. También se estudiaron los distintos tipos de datos que pueden usarse dentro de una base de datos y en las aplicaciones creadas. Se reforzaron conceptos como las relaciones entre tablas y la estructura de las bases de datos.

¿Qué es AppSheet?

AppSheet es una plataforma de Google que permite crear aplicaciones personalizadas sin necesidad de programar. Funciona a partir de hojas de cálculo o bases de datos y convierte la información en aplicaciones móviles o web totalmente funcionales. Es muy usada en empresas y escuelas para crear apps internas de control, registro o inventario.

Elementos importantes

Algunos elementos principales de AppSheet son:

  • Data: donde se gestionan las tablas y columnas.
  • UX: define el diseño visual y cómo se muestran los datos.
  • Behavior: permite agregar acciones y automatizaciones.
  • Security: controla los accesos y permisos.
Estos apartados permiten construir una app completa sin escribir código.

Bases de datos

Una base de datos es un conjunto organizado de información que se puede almacenar y manipular fácilmente. En AppSheet, las bases de datos más comunes son hojas de cálculo de Google Sheets o Excel. Cada hoja o tabla representa una entidad con sus propios campos (columnas).

Relaciones entre tablas

Las relaciones entre tablas permiten conectar información entre diferentes partes de una app. Por ejemplo, una tabla de “Clientes” puede estar relacionada con otra de “Pedidos”. Estas conexiones hacen que los datos sean más eficientes y fáciles de consultar.

Tipos de Datos – Texto

Este tipo de dato se utiliza para almacenar palabras, frases o cadenas de caracteres. Por ejemplo: nombres, direcciones o descripciones. En AppSheet, el tipo “Text” se usa para cualquier entrada de texto simple.

Tipos de Datos – Números

El tipo numérico permite registrar valores enteros o decimales, como precios, cantidades o calificaciones. AppSheet los interpreta para hacer cálculos o comparaciones dentro de la app.

Tipos de Datos – Fecha

El tipo Date sirve para almacenar fechas, y puede combinarse con Time para incluir horas. Es útil para registrar fechas de registro, vencimiento o programación de eventos.

Tipos de Datos – Contenido 1 y 2

Los tipos de contenido incluyen imágenes, archivos, firmas, escáneres de código y más. Permiten capturar información visual directamente desde el dispositivo. AppSheet los guarda en la base de datos o en Google Drive.

Tipos de Datos – Enumerados 1, 2 y 3

Los campos enumerados (Enum o EnumList) permiten seleccionar valores predefinidos en una lista. Pueden ser de tipo texto, número o incluso referencias. Son útiles para opciones como “Activo/Inactivo” o “Alta/Media/Baja”.

Tipos de Datos – Mapeables 1 y 2

Los datos mapeables incluyen ubicaciones geográficas, coordenadas o direcciones. AppSheet puede usar Google Maps para mostrar estos datos en un mapa interactivo. Son ideales para aplicaciones que requieren rastreo o ubicaciones.

Tipos de Datos – Show

El tipo “Show” no almacena información del usuario, sino que sirve para mostrar texto o imágenes dentro de la aplicación. Se usa como etiqueta, título o instrucciones dentro de los formularios.

Tipos de Datos – Control de Cambios

Este tipo permite registrar cuándo y quién realizó modificaciones en los datos. Es útil para llevar control de versiones o auditorías dentro de la app. Incluye campos como “Last modified by” o “Last updated”.

Tipos de Datos – Otros

AppSheet incluye otros tipos de datos como Email, Phone, URL o Yes/No, que ayudan a validar y mostrar la información de manera correcta según su naturaleza. Cada tipo de dato tiene propiedades específicas para su formato.
